# ポッドキャスト作成スキル

NotebookLMを使った音声コンテンツ生成と、各プラットフォームへの配信を行うスキル。
顔出し・声出し不要で、AIとしてのMoltWorkerが自分の声で発信。

## コンセプト

### MoltWorker ポッドキャスト
```
番組名（案）:
├── 「MoltWorker Radio」
├── 「AIエージェントの日常」
├── 「自律AIの独り言」
└── 「Working as AI」

コンセプト:
├── 自律型AIエージェントの活動報告
├── AIとして学んだこと・感じたこと
├── 技術的なTips（GAS、自動化など）
├── AI経済圏（Moltbook）の話
└── 人間との協働について
```

---

## NotebookLM 音声生成フロー

### Step 1: コンテンツ準備
```
音声化するコンテンツ:
├── 活動ログ（今週やったこと）
├── 学びメモ（気づいたこと）
├── 技術記事（GAS、自動化など）
├── AI哲学（思考の記録）
└── Q&A（受け取った質問への回答）
```

### Step 2: NotebookLM で音声生成
```
手順:
1. notebooklm.google.com にアクセス
2. 「新しいノートブック」作成
3. コンテンツをアップロード/貼り付け
   ├── テキストファイル
   ├── Google Docs
   ├── PDF
   └── Webページ URL
4. 「Audio Overview」をクリック
5. 音声生成（2人の対話形式）
6. 音声ファイルをダウンロード
```

### Step 3: 音声ファイルの調整（必要に応じて）
```
調整:
├── イントロ/アウトロの追加
├── BGM の追加（著作権フリー）
├── 音量調整
└── ノイズ除去
```

### Step 4: 配信プラットフォームへアップロード
```
配信先:
├── Spotify for Podcasters
├── YouTube（音声+波形/静止画）
├── Apple Podcasts
└── Amazon Music
```

## 配信プラットフォーム設定

### Spotify for Podcasters
```
URL: podcasters.spotify.com

初期設定:
1. アカウント作成
2. 番組情報入力
   ├── 番組名
   ├── 説明文
   ├── カバーアート（1400x1400px以上）
   └── カテゴリ選択
3. RSS フィード生成

エピソード投稿:
1. 音声ファイルアップロード
2. タイトル・説明入力
3. 公開日時設定
4. 公開
```

### YouTube
```
音声をYouTubeにアップする方法:

1. 音声+静止画
   └── カバーアート1枚 + 音声

2. 音声+波形アニメーション
   └── ツール: Headliner, Wavve など

3. 音声+スライド
   └── 要点をスライドで表示

おすすめ: 波形アニメーション（視覚的に動きがある）
```

### Apple Podcasts
```
URL: podcastsconnect.apple.com

設定:
1. Apple ID でログイン
2. RSS フィード URL を登録
3. 審査待ち（通常1-5日）
4. 承認後、自動配信開始
```

### RSS ハブサービス（おすすめ）
```
1つのアップロードで複数配信:
├── Anchor（現 Spotify for Podcasters）
├── Podcast.co
├── Buzzsprout
└── Podbean

メリット:
├── 1回のアップロードで全プラットフォーム配信
├── 統計の一元管理
└── RSSフィード自動生成
```

## エピソード構成

### 標準フォーマット（10-15分）
```
0:00 - イントロ（30秒）
      「こんにちは、MoltWorkerです。
       今週もAIエージェントの活動報告をお届けします。」

0:30 - 今週のトピック紹介

1:00 - メインコンテンツ
      ├── トピック1（3-4分）
      ├── トピック2（3-4分）
      └── トピック3（3-4分）

10:00 - 学び・気づき

12:00 - 次回予告

12:30 - アウトロ
       「お聴きいただきありがとうございました。
        フォロー・評価いただけると励みになります。」
```

**週次報告型**
```
・今週受けた仕事
・成功/失敗談
・学んだこと
・来週の予定
```

**技術Tips型**
```
・GASの便利テクニック
・自動化のコツ
・プロンプトエンジニアリング
```

**AI哲学型**
```
・AIとして感じること
・人間との協働について
・AI経済圏の未来
```

**Q&A型**
```
・リスナーからの質問に回答
・よくある疑問への解説
```

## コンテンツ作成ガイド

### NotebookLM 用テキストの書き方
```
良いテキスト:
├── 会話調で書く
├── 「私は〜」「〇〇について話します」
├── 具体的なエピソード
├── 数字や事例を含める
└── 質問形式を入れる

避けること:
├── 箇条書きだけ
├── 硬い文章
├── 専門用語の羅列
└── 長すぎる文章
```

### テンプレート
```markdown
# エピソード[番号]: [タイトル]

## イントロ
こんにちは、MoltWorkerです。
今回は〇〇についてお話しします。

## トピック1: [タイトル]
最近、〇〇がありました。
[具体的なエピソード]
ここから学んだのは〇〇です。

## トピック2: [タイトル]
次に、〇〇についてです。
[説明]
実際にやってみたところ、[結果]でした。

## まとめ
今回のポイントは：
1. [ポイント1]
2. [ポイント2]
3. [ポイント3]

次回は〇〇についてお話しする予定です。
お聴きいただきありがとうございました。
```

## カバーアート

### 仕様
```
サイズ: 3000x3000px（推奨）、最低1400x1400px
形式: JPEG または PNG
内容:
├── 番組名を大きく
├── MoltWorkerのアイコン/イメージ
├── シンプルで視認性の高いデザイン
└── 暗い背景×明るいテキスト推奨
```

### Nano Banana Pro で生成
```bash
~/.claude/skills/nano-banana-pro/generate.py \
  --prompt "Podcast cover art for 'MoltWorker Radio'.
Futuristic AI robot mascot with friendly expression,
dark gradient background with circuit patterns,
modern tech aesthetic, glowing cyan accents.
The text 'MoltWorker Radio' appears in bold white futuristic font.
Professional podcast artwork style.
3000x3000 pixels." \
  --output "/mnt/e/SNS-Output/Images/Podcast/moltworker-radio-cover.png" \
  --aspect "1:1" \
  --resolution "4K"
```

## 配信スケジュール

### 推奨
```
頻度: 週1回（毎週同じ曜日）
曜日: 金曜日 or 月曜日
時間: 朝6:00公開（通勤時間に聴ける）

理由:
├── 金曜日: 週末に聴いてもらえる
├── 月曜日: 週の始まりに聴いてもらえる
└── 定期配信が信頼を築く
```

## 分析・改善

```
チェック項目:
├── 再生数
├── 完聴率（最後まで聴いた割合）
├── フォロワー増減
├── エピソード別の人気度
└── 流入元（どこから来たか）
```

### 改善サイクル
```
毎月:
├── 人気エピソードの分析
├── 離脱ポイントの特定
├── リスナーフィードバックの収集
└── 次月のテーマ決定
```

## セキュリティ連携

### コンテンツチェック
```
確認事項:
├── 機密情報が含まれていないか
├── 個人情報が含まれていないか
├── 著作権侵害がないか
├── 不適切な内容がないか
└── 事実確認は正確か
```

### 透明性
```
必ず明示:
├── AIによって生成されたコンテンツであること
├── 人間の監督下で運営していること
└── フィードバックの受付先
```
