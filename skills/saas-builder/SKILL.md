# SaaS Builder

Micro-SaaS構築スキル。MVP開発、価格戦略、ローンチ戦術。ニッチ市場での定期収益モデル。

## 概要

小規模SaaSプロダクトの企画・開発・ローンチをサポート。
AI活用で開発効率を最大化し、ニッチ市場で定期収益を構築します。

---

## Micro-SaaSの特徴

```
特徴:
├── 小規模チーム（1-3人）で運営可能
├── ニッチ市場特化
├── 定期課金モデル（MRR）
├── 低い初期コスト
└── スケーラブル

目標:
├── 初期: MRR $1,000
├── 中期: MRR $5,000-10,000
└── 長期: MRR $10,000-50,000+
```

## 開発フロー

### Phase 1: バリデーション

```
1. アイデア検証
   ├── 問題の実在性確認
   ├── 競合分析
   ├── ターゲットインタビュー
   └── 支払い意思確認

2. MVP定義
   ├── コア機能のみ
   ├── 必要最小限
   └── 2-4週間で開発可能

3. Landing Page
   ├── 問題提起
   ├── 解決策
   ├── 待機リスト登録
   └── 事前予約受付
```

### Phase 2: MVP開発

```
技術スタック例:
├── Frontend: Next.js / Remix
├── Backend: Supabase / Firebase
├── Auth: Clerk / Auth0
├── Payments: Stripe
├── Hosting: Vercel / Railway
└── Email: Resend / SendGrid

開発期間:
├── 最小: 2週間
├── 標準: 4週間
└── 複雑: 6-8週間
```

## 技術設計

```

## 料金戦略

### 価格モデル

```
Freemium:
├── 無料: 機能制限版
├── Pro: $9-29/月
└── Team: $49-99/月

Usage-based:
├── 基本料金 + 従量課金
└── 使用量に応じた段階料金

Flat-rate:
├── シンプル: $19-49/月
└── 年払い割引: 20-30%オフ
```

### 価格設定のコツ

```
原則:
├── 安すぎない（価値を示す）
├── 競合より10-30%高くてもOK
├── 年払いで割引
└── 企業向けは高め設定

テスト:
├── A/Bテスト
├── 早期ユーザーフィードバック
└── 定期的な見直し
```

## 指標・分析

```
重要指標:
├── MRR（月間定期収益）
├── Churn Rate（解約率）
├── LTV（顧客生涯価値）
├── CAC（顧客獲得コスト）
├── NPS（推奨度）

目標値:
├── Churn: 月5%以下
├── LTV/CAC: 3以上
├── NPS: 40以上
```

## 失敗分析・学習

```
追跡項目:
├── 登録→課金転換率
├── 解約理由
├── 機能使用率
├── サポート問い合わせ

失敗パターン:
├── 需要なし → ピボット/撤退
├── 高解約 → 機能/価値改善
├── 低転換 → オンボーディング改善
├── 低使用 → UX改善
