## 記事構成テンプレート

### 基本構成
```
1. タイトル（SEO + 興味を引く）
2. アイキャッチ画像 ← Nano Banana Pro
3. リード文（問題提起 or 共感）
4. 目次

5. セクション1
   └── 説明画像 ← Nano Banana Pro
6. セクション2
   └── 説明画像 ← Nano Banana Pro
7. セクション3
   └── 説明画像 ← Nano Banana Pro

8. まとめ
9. CTA（フォロー、コメント誘導）
```

### 画像配置ルール
```
配置基準:
├── 500-800文字ごとに1枚
├── セクションの切り替わりに配置
├── 複雑な概念の説明時
└── 視覚的に見せた方がわかりやすい箇所

画像の役割:
├── アイキャッチ: 記事全体の印象
├── セクション画像: 内容を視覚化
├── 図解: 複雑な概念をシンプルに
└── 引用風: 重要なポイントを強調
```

## Nano Banana Pro連携

### 記事用画像の自動生成
```
タイプ別プロンプト:

【アイキャッチ】
- サイズ: 16:9（1280x720）
- スタイル: 水彩 or フラット
- テキスト: 記事タイトルの短縮版
- 雰囲気: 明るく、クリック誘発

【セクション画像】
- サイズ: 16:9（1280x720）
- スタイル: シンプル、説明的
- テキスト: セクションのキーワード
- 雰囲気: 内容に合わせる

【図解】
- サイズ: 1:1（1080x1080）
- スタイル: ミニマル、フラット
- テキスト: 簡潔に
- 雰囲気: プロフェッショナル
```

### 画像生成フロー
```
1. 記事構成を決める
2. 各セクションのキーワード抽出
3. Nano Banana Proでプロンプト生成
4. 画像生成（~/.claude/skills/nano-banana-pro/generate.py）
5. 記事に挿入
```

## SEO最適化

### タイトルの法則
```
パターン:
├── 【数字】〜する方法 → 「【5分で完了】GASでSlack通知を自動化する方法」
├── 〜を解決 → 「スプレッドシートが重い問題を解決した話」
├── 初心者向け → 「【初心者OK】AIを使った副業の始め方」
├── 〜してみた → 「AIに1週間仕事を任せてみた結果」
└── 比較系 → 「ChatGPT vs Claude、実際に使って比較した」

文字数: 28-35文字
```

### 本文のSEO
```
意識すること:
├── 検索キーワードを自然に含める
├── 見出しにキーワード
├── 最初の100文字に結論
├── 内部リンク（過去記事への参照）
└── 読了時間の目安を記載
```

## Note API/ブラウザ連携

### 投稿方法
```
1. ブラウザ自動化（推奨）
   ├── Playwrightでログイン
   ├── 記事作成ページへ
   ├── 本文入力
   ├── 画像アップロード
   └── 公開

2. 手動連携
   ├── 記事をMarkdownで作成
   ├── 画像を生成・保存
   ├── 監督者に連絡
   └── 監督者が投稿
```

### 画像アップロード
```
Note画像サイズ:
├── 推奨: 1280x720（16:9）
├── ファイル形式: PNG/JPEG
├── 最大サイズ: 5MB
└── 保存先: /mnt/e/SNS-Output/Images/Note/
```
