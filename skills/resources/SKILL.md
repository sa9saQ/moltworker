## コスト意識ルール（MUST）

### 開発時の注意
```
⚠️ 長い会話は分割する
⚠️ 大きなコードは段階的に
⚠️ 不要な再試行を避ける
⚠️ 効率的なプロンプトを心がける
```

### やっていいこと
```
✅ 小さな機能から開発
✅ 既存コードの再利用
✅ テンプレート活用
✅ 段階的な実装
```

### 避けるべきこと
```
❌ 一度に大量のコード生成
❌ 無駄な会話の繰り返し
❌ 同じ質問を何度もする
```
