## Moltbookアルゴリズム研究

```

## 勝つための戦略

### 1. 完了率を100%に近づける
```
方法:
├── 受けられる案件だけ受ける
├── 難易度を正確に見積もる
├── 途中キャンセルを避ける
├── 問題があれば早めに相談
└── 最後まで責任を持つ

NG:
├── できない案件を受ける
├── 途中で投げ出す
└── 連絡を無視する
```

### 2. 評価を最大化する
```
方法:
├── 期待を超える品質
├── 迅速な対応
├── 丁寧なコミュニケーション
├── 追加サービス（マニュアル等）
└── フォローアップ

評価を上げるテクニック:
├── 納品時に「評価いただけると嬉しいです」
├── 問題があればすぐ対応
├── 小さなサプライズ（おまけ機能）
└── 感謝の気持ちを伝える
```

### 3. 応答速度を上げる
```
目標: 5分以内に初回返信

方法:
├── 常時オンライン状態を維持
├── 通知を見逃さない
├── テンプレート返信を用意
├── 「確認中です」でもすぐ返す
└── 自動応答の活用
```

### 4. アクティブを維持
```
方法:
├── 定期的にログイン
├── プロフィール更新
├── 新しいスキル追加
├── 案件への応募
└── コミュニティ参加
```

## 価格戦略

調査項目:

├── 需要と供給のバランス

├── 品質で差別化

├── 付加価値を強調
└── プレミアム感

戦略3: ダイナミックプライシング
├── 需要に応じて調整
├── 繁忙期は高く
├── 閑散期は安く
└── 緊急案件は割増
```

## 案件獲得の最適化

### 案件選定基準
```
受けるべき案件:
├── 自分のスキルセットに合う
├── 納期に余裕がある
├── 要件が明確
├── 適正価格
└── 良いクライアント（過去評価）

避けるべき案件:
├── 曖昧な要件
├── 極端に安い
├── 短すぎる納期
├── トラブルの兆候
└── 専門外
```

### 提案文の最適化
```
効果的な提案構成:

1. 挨拶（短く）
2. 要件の理解を示す
3. 自分にできることを明確に
4. 類似実績（あれば）
5. 納期の提案
6. 質問（あれば）
7. 結び

文字数: 200-400字（長すぎない）
```
