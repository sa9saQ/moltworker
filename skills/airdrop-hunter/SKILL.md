# Airdrop Hunter

仮想通貨エアドロップの追跡・請求自動化。

## 概要

新規プロジェクトのトークン配布（エアドロップ）を追跡し、
適格性確認と請求を自動化。

## 機能

### エアドロップ追跡
- 新規エアドロップ情報収集
- 適格性チェック
- 締切管理
- 価値評価

### 自動請求
- ガス代最適化
- バッチ請求
- 複数ウォレット管理

### 分析
- 過去エアドロップ統計
- 期待価値計算
- ポートフォリオ追跡

## 使用方法

### 適格性確認
```
「[ウォレットアドレス]のエアドロップ適格性を確認」
「今週の新規エアドロップをリスト」
```

### 請求
```
「[プロジェクト名]のエアドロップを請求」
「全ての保留中エアドロップを請求」
```

### 追跡設定
```
「[プロジェクト]のエアドロップを追跡開始」
「エアドロップカレンダーを表示」
```

## エアドロップタイプ

### Retroactive Airdrop
```
過去の利用履歴に基づく配布:
- プロトコル使用者
- ガバナンス参加者
- 早期採用者
```

### Holder Airdrop
```
特定トークン保有者への配布:
- スナップショット時点の保有者
- 最低保有量条件あり
```

### Task-Based Airdrop
```
タスク完了による配布:
- SNSフォロー
- テストネット使用
- コミュニティ参加
```

## 戦略

### ファーミング戦略
```
1. 有望プロジェクトの特定
   - VCバック確認
   - TVL/ユーザー成長
   - チーム背景

2. 適格性獲得活動
   - テストネット使用
   - メインネット取引
   - ガバナンス参加
   - 流動性提供

3. 記録管理
   - ウォレット別活動記録
   - 取引ハッシュ保存
   - スナップショット日管理
```

### コスト管理
```
- ガス代と期待価値のバランス
- バッチ処理でガス節約
- 低ガス時間帯を狙う
- L2/サイドチェーン活用
```

## リスク管理

### 詐欺対策
```
⚠️ 公式ソースのみ信頼
⚠️ シードフレーズ入力要求は詐欺
⚠️ 不明なコントラクトは承認しない
⚠️ 「請求前に送金」は詐欺
```

### ウォレット分離
```
メインウォレット: 資産保管用
├── 高額資産のみ
└── 不要な承認なし

ファーミングウォレット: 活動用
├── 少額のみ
├── 定期的に新規作成
└── 承認は最小限
```
